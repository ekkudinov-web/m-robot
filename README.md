# Минфин-робот

Android-приложение для автоматической торговли фьючерсами USD/RUB, CNY/RUB,
EUR/RUB на Мосбирже по публикации Минфина РФ о ежедневном объёме операций
бюджетного правила. Сделано по архитектуре приложения «Робот ключевой ставки»
(декомпилированный APK был отправной точкой).

## Что делает

1. Ждёт начала окна 11:30 МСК в третий рабочий день месяца (вычисляется автоматически).
2. Каждые 2 секунды параллельно опрашивает `minfin.gov.ru/ru/press-center/`
   (основной источник) и `tass.ru/ekonomika` (страховка от блокировки).
3. Находит новую публикацию о нефтегазовых доходах по ключевым словам в заголовке.
4. Парсит число ежедневного объёма операций (₽ млрд, знаковое: + покупка, − продажа).
5. Сравнивает с заданными пользователем сценариями (>X или <X).
6. Параллельно отправляет ордера на фьючерсы через T-Invest API.
7. При паузе операций ("не проводить операций…") — НЕ торгует ни при каких условиях.

## Архитектура

```
data/
  minfin/    основной источник (minfin.gov.ru)
  tass/      fallback на ТАСС
  tbank/     T-Invest клиент (Sandbox + Production)
  settings/  EncryptedSharedPreferences для токенов
  log/       in-memory ring buffer 500 записей для UI
domain/
  model/     Scenario, RobotConfig, MinfinPublication
  engine/    ScenarioEvaluator, PriceRoundingUtil, BusinessDays
  usecase/   StartRobotUseCase — главный оркестратор
background/  RobotExecutionService (foreground service)
ui/          Compose: MainScreen, SettingsScreen, LogsScreen
```

## Сборка

1. Открыть проект в Android Studio (Hedgehog 2023.1.1+).
2. Дождаться Gradle Sync.
3. Build → Make Project.
4. Run на физическом устройстве (НЕ эмуляторе — там нет российских IP).

Зависимости (см. `app/build.gradle.kts`):
- Kotlin 1.9.22, Compose BOM 2024.02.00
- Retrofit 2.9.0 + OkHttp 4.12.0 + kotlinx.serialization 1.6.2
- androidx.security:security-crypto для EncryptedSharedPreferences

## Получение токенов T-Invest

**Sandbox** (рекомендуется для первого теста):
1. Зайти на `https://www.tinkoff.ru/invest/settings/api/`
2. Создать токен типа "Sandbox", дать ему имя, скопировать.
3. **Создать sandbox-счёт** через REST (один раз):
   ```bash
   curl -X POST https://sandbox-invest-public-api.tinkoff.ru/rest/tinkoff.public.invest.api.contract.v1.SandboxService/OpenSandboxAccount \
     -H "Authorization: Bearer <YOUR_SANDBOX_TOKEN>" \
     -H "Content-Type: application/json" \
     -d "{}"
   ```
4. Пополнить sandbox-счёт через `SandboxService/SandboxPayIn`.
5. В приложении: вкладка "Настройки" → "Sandbox-токен" → вставить → Сохранить.

**Production** (реальные деньги):
1. На том же сайте создать токен **с правами на торговлю**.
2. В приложении: вкладка "Настройки" → переключатель режима → подтвердить переход
   в боевой режим в диалоге → "Production-токен" → вставить → Сохранить.

Токены хранятся **локально** в `EncryptedSharedPreferences` (AES-256-GCM,
мастер-ключ в Android Keystore).

Парсер протестирован на 9 реальных публикациях Минфина за 2020–2026 годы
(см. сообщения чата для исходных данных). Корректно распознаёт:

| Дата | Тип | Ожидаемое | Получено |
|---|---|---|---|
| 15.01.2020 | покупка | +18.2 | +18.2 ✓ |
| 05.12.2023 | покупка | +11.7 | +11.7 ✓ |
| 15.01.2024 | продажа | −4.1  | −4.1 ✓ |
| 05.11.2024 | покупка | +4.2  | +4.2 ✓ |
| 03.04.2025 | продажа | −1.6  | −1.6 ✓ |
| 07.05.2025 | покупка | +2.3  | +2.3 ✓ |
| 03.12.2025 | продажа | −5.6  | −5.6 ✓ |
| 04.03.2026 | ПАУЗА   |  0.0  |  0.0 ✓ |
| 14.05.2026 | покупка | +5.8  | +5.8 ✓ |

Округление лимитной цены — 12/12 тестов, включая edge case `10.450 / шаг 0.001`
(без epsilon-tolerance давал бы 10.449).

## Ожидаемая публикация Минфина

3-й рабочий день месяца, обычно ~12:00 МСК. Приложение начинает мониторинг
с 11:30 МСК и продолжает до 14:00 МСК. Если в окне публикация не появилась —
СТОП с уведомлением (вручную перезапустите на следующий день).

После 18 июня 2026 г. (экспирация Si-6.26, CR-6.26, Eu-6.26) — приложение
требует перенастройки на новый контракт. Сценарии сбрасываются при перезапуске.

## Известные ограничения

1. **Российский IP обязателен.** Сайт minfin.gov.ru блокирует датацентровые IP
   (проверено: с Frankfurt-сервера возвращает 503 "Доступ временно ограничен").
   Если телефон под VPN → отключить перед запуском.
2. **Не учитываются российские праздники** при вычислении 3-го рабочего дня —
   только субботы/воскресенья. Если 3-й рабочий день попадает на майские/новогодние
   каникулы — задать дату вручную в UI.
3. **Парсер работает только пока Минфин не сменит формулировку.** Текущая
   формулировка стабильна с 2020 года, но если изменится — потребуется обновить
   regex в `MinfinParser.kt`.
4. **Нет проверки достаточности ГО** перед ордером — пользователь сам отвечает
   за наличие денег на счёте. При недостатке T-Invest вернёт ошибку, которая
   попадёт в лог.
5. **Нет защиты от проскальзывания** при market-ордере. Если важно — задать
   лимитную цену в сценарии.

## Дисклеймер
